hexo+github搭建博客

发表于 2022-02-05

搭建教程 Hexo文档 Next文档

1 安装 Node. Js

首先下载稳定版Node.js，安装选项全部默认，一路点击 Next。

最后安装好之后，按 Win+R 打开命令提示符，输入 node -v 和 npm -v，如果出现版本号，那么就安装成功了。

2 安装 Hexo

在合适的地方新建一个文件夹，用来存放自己的博客文件，比如我的博客文件都存放在 E:\obsidian\blog 目录下。

在该目录下右键点击 Git Bash Here，打开 git 的控制台窗口，以后我们所有的操作都在 git 控制台进行，就不要用 Windows 自带的控制台了。

定位到该目录下，输入 npm i hexo-cli -g 安装 Hexo。会有几个报错，无视它就行。安装完后输入 hexo -v 验证是否安装成功。

然后初始化我们的网站，输入 hexo init 初始化文件夹，接着输入 npm install 安装必备的组件。

这样本地的网站配置也弄好啦，输入 hexo g 生成静态网页，然后输入 hexo s 打开本地服务器，然后浏览器打开http://localhost:4000/，就可以看到我们的博客啦，效果如下：

3 创建一个 git 仓库

  1. 登录 github 官网创建 github 账号： http://www.github.com/
  2. 创建仓库: 仓库名字必须是 abc. Github. Io ，abc 是注册的 github 账号用户名
  3. 创建成功后点击 Settings，在该页面左侧点击 Pages，点击 Choose a theme 选择一个主题。然后等一会儿，返回到 Pages，点击链接可以进入博客主页

4 连接 Github 与本地

  1. 配置 ssh

使用 ssh 进行连接，若无 ssh 密匙，则进行配置（教程） 然后输入 ssh -T git@github.com，如果出现 Hi xxx! You've successfully authenticated, but GitHub does not provide shell access. 说明配置成功；已有 ssh 则直接进行验证，不需要配置。

  1. 修改 _config.yml 文件

打开根目录下的 _config.yml 文件，修改最后一行的配置。repository 要用 ssh 的链接，token 可以不加，所有 : 后要有空格

  1. 部署博客
    首先在博客根目录下右键打开 git bash，安装一个扩展 npm i hexo-deployer-git。然后输入 hexo deploy 完成部署

5 文章的发布

所有要发布博客的原文件（ .md 文件）都要存在 ...\blog\source\_posts 目录下。

因为使用图床所以不用存图片等资源文件在本地，所以可以使用其它 markdown 编辑器创建并编辑 md 文件，完成后再将 md 文件移到该目录下即可。然后按顺序执行以下操作即可发布博客

  1. 在根目录下输入 hexo g 生成静态网页
  2. 输入 hexo s 可以本地预览效果
  3. 输入 hexo d 上传到 github 上

这时打开 github. Io 主页就能看到发布的文章啦。

6 其它 hexo 命令

  1. hexo n "学习笔记 六" : 新建一篇标题为 学习笔记 六 的文章，因为标题里有空格，所以加上了引号。
  2. hexo clean : 清除缓存文件 db.json 和已生成的静态文件 public。在文章发布前最好都加上
